# Contributing Guidelines

Thanks for taking the time to contribute to **Awesome PineTime**! 🎉

This is a curated list, so quality and relevance matter more than quantity.
Please read these guidelines before opening a pull request.

## How to add an entry

1. **Fork** this repository and create a branch for your change.
2. Add your entry to the most appropriate section in [`README.md`](README.md), keeping
   entries **alphabetically ordered within their section** where it makes sense
   (the flagship/most-used project may be listed first).
3. Use the following format:

   ```markdown
   - [Project name](https://link) - Short, factual description of what it is and why it's useful. `License`
   ```

   - Start the description with a capital letter and end it with a period.
   - Keep it to one sentence - say what it *is*, not marketing copy.
   - Include the software license in backticks at the end for code projects.
4. If you add a new section, also add it to the **Contents** table at the top.
5. Open a **pull request** with a clear title and a one-line note on why the entry
   is awesome.

## What belongs here

- Open-source firmware, operating systems and forks that run on the PineTime.
- Emulators, simulators and development tooling.
- Companion apps and integrations.
- High-quality documentation, datasheets, schematics and tutorials.
- Official and reputable places to buy the hardware.

## What does not belong here

- Abandoned or non-functional projects (unless of clear historical value).
- Duplicate entries or links that don't add anything new.
- Self-promotion of low-quality or unrelated material.
- Broken links - please check your URLs before submitting.

## Pull request checklist

- [ ] The entry is in the correct section.
- [ ] The link works and points to the canonical source.
- [ ] The description is concise, factual and free of typos.
- [ ] A license is included for code projects.
- [ ] The **Contents** list is updated if a section was added.
